# Phase 14 — License Persistence & Expiration Parsing Fix

## المشكلة
كانت صفحة الترخيص تعرض: `تاريخ انتهاء الترخيص غير مفهوم`، وهذا يجعل التطبيق يطلب التفعيل عند كل تشغيل حتى لو كان ملف الترخيص موجودًا.

## السبب
1. فحص الترخيص كان يرفض أي قيمة انتهاء غير ISO فقط، بينما خدمة الترخيص قد ترسل قيمًا مثل `lifetime`, `unlimited`, `never`, `غير محدود` أو timestamp.
2. ملفات الترخيص كانت تُكتب داخل مجلد التثبيت/السورس، وهذا غير مناسب لنسخ Windows أو Program Files أو PyInstaller.

## الإصلاح
- نقل ملفات الترخيص إلى مجلد إعدادات قابل للكتابة:
  `%APPDATA%/Hawaa/config/license.dat`
  `%APPDATA%/Hawaa/config/network_license.dat`
- دعم قراءة الملفات القديمة من مجلد المشروع وترحيلها تلقائيًا للمسار الجديد.
- دعم صيغ انتهاء متعددة: ISO, YYYY-MM-DD, DD/MM/YYYY, DD.MM.YYYY, Unix timestamp seconds/ms, lifetime/unlimited/permanent/غير محدود.
- عدم إسقاط البرنامج إذا فشل ترحيل الملف القديم.

## النتيجة
إذا كان الترخيص دائمًا أو تاريخ الانتهاء بصيغة مختلفة، لن يطلب البرنامج التفعيل في كل تشغيل.
