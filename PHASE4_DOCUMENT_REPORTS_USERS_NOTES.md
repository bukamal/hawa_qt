# Phase 4 — Document Reports + Inline User Management

## الهدف
استكمال التحويل من نوافذ/Widgets قديمة إلى Document Shell + Inline Panels، مع تثبيت التقارير وإدارة المستخدمين فوق طبقات خدمات مركزية.

## ما تغيّر

### 1) إدارة المستخدمين
- إضافة `services/user_service.py` كطبقة مركزية لإدارة المستخدمين.
- منع أي عملية إدارة مستخدمين لغير المدير من الخدمة نفسها، وليس من الواجهة فقط.
- حماية المدير الأساسي `id=1` من الحذف أو تخفيض الصلاحية.
- منع حذف آخر مدير في النظام.
- منع حذف المستخدم الحالي أثناء الجلسة.
- دعم أدوار أوسع:
  - `admin`
  - `manager`
  - `accountant`
  - `auditor`
  - `user`
  - `viewer`

### 2) واجهة مستخدمين Inline
- إضافة `ui/documents/users_document.py`.
- إضافة `ui/editors/user_editor_panel.py`.
- إضافة `ui/shell/users_shell.py`.
- استبدال صفحة المستخدمين في `views/main_window.py` بـ `UsersShell` بدل `UsersWidget`.
- إصلاح عيب قديم في `UsersWidget`: لم يعد يتم إخفاء أول عمود ظاهر؛ لأن `id` موجود داخل `key_fields` وليس كعمود مرئي.

### 3) التقارير
- إضافة `services/report_service.py` كمصدر واحد لحساب التقارير.
- إضافة `ui/documents/reports_document.py`.
- إضافة `ui/shell/reports_shell.py`.
- ربط صفحة التقارير فعليًا في الشريط الجانبي الرئيسي.
- إضافة معاينة طباعة Inline عبر `PrintPreviewPanel` و`PrintService`.

### 4) العملات والتقارير
- التقارير تعتمد على `amount_base` بالدولار كأساس محاسبي.
- `display_currency` تبقى للعرض فقط.
- لا يتم إعادة تسعير القيود التاريخية.
- عند تغيير سعر الصرف، تُحدّث طريقة عرض التقارير، لكن أصل القيد يبقى محافظًا على سعره التاريخي.

### 5) الربط بين الصفحات
- عند تغيير قيد في الحسابات، يتم تحديث التقارير.
- عند تغيير أسعار الصرف في الإعدادات، يتم تحديث الحسابات والتقارير.

### 6) الاختبارات
- إضافة `tests/conftest.py` لتوفير Stub صغير لـ `QSettings` عند غياب PyQt في بيئة الاختبار.
- إضافة اختبارات لإدارة المستخدمين.
- إضافة اختبار لخدمة التقارير.
- إجمالي الاختبارات الآن: 7.
- نتيجة الفحص:
  - `python -m compileall -q .` ✅
  - `PYTHONPATH=. pytest -q` ✅ 7 passed

## ملاحظات متبقية
- تغيير كلمة مرور مستخدم آخر ما زال يحتاج تحويلًا لاحقًا إلى Inline Security Panel.
- `SettingsWidget` و`AuditLogWidget` ما زالا Widgets قديمة.
- الخطوة التالية المنطقية: تحويل الإعدادات إلى `SettingsDocument` وتقسيمها إلى Documents فرعية: العملات، النسخ الاحتياطي، الشبكة، الترخيص، المظهر.
